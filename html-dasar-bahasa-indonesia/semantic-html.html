#11. Semantic HTML

Pada bab-bab sebelumnya, kita telah mempelajari berbagai elemen HTML untuk menampilkan teks, gambar, tabel, formulir, hingga multimedia. Seluruh elemen tersebut dapat digunakan untuk membangun sebuah halaman web.

Namun, jika sebuah website hanya menggunakan elemen seperti <div> untuk membungkus seluruh isi halaman, maka browser maupun mesin pencari akan kesulitan memahami bagian-bagian penting dari halaman tersebut.

HTML5 memperkenalkan Semantic HTML, yaitu sekumpulan elemen yang memiliki makna tertentu sehingga struktur halaman menjadi lebih jelas, mudah dipahami, dan lebih teroganisir.



11.1. Apa itu Semantic HTML ?

Semantic HTML adalah penggunaan elemen HTML yang memiliki arti atau makna sesuai dengan isi yang dibungkus. Sebagai contoh:

  • <header> berarti bagian kepala halaman
  • <nav> berarti navigasi
  • <main> berarti isi utama halaman
  • <footer> berarti bagian bawah halaman

Dengan menggunakan Semantic HTML, kode menjadi lebih mudah dipahami baik oleh manusia maupun oleh browser.



11.2. Mengapa Semantic HTML Penting ?

Penggunaan Semantic HTML memberikan banyak keuntungan dalam pengembangan website modern.

1. Struktur halaman lebih jelas

Setiap bagian halaman memiliki elemen yang sesuai dengan fungsinya. Contoh:

Header
↓
Navigasi
↓
Konten Utama
↓
Sidebar
↓
Footer

Struktur seperti ini lebih mudah dipahami dibandingkan hanya menggunakan banyak elemen <div>.

2. Mempermudah membaca kode

Perhatikan conoth berikut ketika menggunakan <div> :

<div>
    ...
</div>

<div>
    ...
</div>

<div>
    ...
</div>

Sulit mengetahui fungsi masing-masing bagian. Sedangkan ketika menggunakan Semantic HTML:

<header>

</header>

<nav>

</nav>

<main>

</main>

<footer>

</footer>

Dari nama elemennya saja kita sudah mengetahui fungsi setiap bagian.

3. Membantu mesin pencari (SEO)

Mesin pencari seperti Google lebih mudah memahami struktur halaman apabila menggunakan elemen semantik. Sebagai contoh:

  • Mana isi utama artikel
  • Mana menu navigasi
  • Mana informasi tambahan
  • Mana bagian penutup

Hal ini dapat membantu proses pengindeksan halaman.

4. Meningkatkan aksesibilitas

Screen reader yang digunakan oleh pengguna berkebutuhan khusus dapat mengenali bagian-bagian halaman dengan lebih baik. Misalnya:

  • Header
  • Navigasi
  • Konten utama
  • Footer

Tanpa harus membaca seluruh halaman dari awal.



11.3. Elemen <header>

Elemen <header> digunakan untuk menampilkan bagian kepala suatu halaman atau suatu bagian tertentu. Biasanya berisi:

  • Logo
  • Judul website
  • Judul artikel
  • Deskripsi singkat

Elemen <header> digunakan pada bagian atas website, bagian atas artkel, ataupun bagian atas section tertentu.



11.4. Elemen <nav>

Elemen <nav> digunakan untuk menampilkan kumpulan tautan navigasi. Biasanya berisi:

  • Menu utama
  • Menu samping
  • Breadcrumb
  • Daftar halaman


11.5. Elemen <main>

Elemen <main> dgunakan untuk membungkus isi utama halaman. Bagian inilah yang menjadi fokus utama website. Misalnya:

  • Artikel
  • Produk
  • Galeri
  • Konten pembelajaran

Dalam satu halaman HTML sebaiknya hanya terdapat satu elemen <main>.



11.6. Elemen <section>

Elemen <section> digunakan untuk mengelompokkan suatu bagian yang memiliki tema tertentu. Misalnya:

  • Profil
  • Layanan
  • Galeri
  • Testimoni

Elemen <section> digunakan untuk membungkus konten yang dapat berdiri sendiri.



11.7. Elemen <article>

Elemen <article> digunalan untuk membungkus konten yang dapat berdiri sendiri. Contohnya:

  • Artikel blog
  • Berita
  • Tutorial
  • Postingan forum

Perbedaan <section> dan <article>

<section> <article>
Mengelompokkan bagian halaman Konten yang berdiri sendiri
Bagian dari halaman Dapat dipublikasi saecara terpisah
Contoh: Profil, Layanan Contoh: Artikel, Berita


11.8. Elemen <aside>

Elemen <aside> sigunakan untuk menampilkan informasi tambahan yang berkaitan dengan isi utama. Biasanya berisi:

  • Artikel populer
  • Kategori
  • Iklan
  • Profil penulis
  • Daftar artkel terbaru


11.9. Elemen <footer>

Elemen <footer> digunakan untuk menampilkan bagian bawah halamn. Biasanya berisi:

  • Copyright
  • Kontak
  • Alamat
  • Media sosial
  • Informasi perusahaan

Biasanya elemen ini diletakkan pada bagian paling bawah halaman. Namun, sebuah artikel juga dapat memiliki footer tersendiri.
