#2. Persiapan Lingkungan Pengembangan

2.1. Mengapa Perlu Mempersiapkan Lingkungan Pengembangan ?

Sebelum mulai menulis kode HTML, seorang web developer perlu menyiapkan lingkungan kerja yang nyaman dan terorganisir. Lingkungan pengembangan yang baik akan memudahkan proses pembuatan, pengujian, dan pemeliharaan website.

Berbeda dengan bahasa pemrograman lain yang membutuhkan proses kompilasi atau instalasi perangkat lunak tambahan, HTML cukup dibuat menggunakan editor teks dan dijalankan melalui browser.

Hal ini menjadikan HTML sebagai salah satu teknologi web yang paling mudah dipelajari oleh pemula.



2.2. Memilih Text Editor

Text editor adalah aplikasi yang digunakan untuk menulis dan mengedit kode program.

Meskipun HTML dapat ditulis menggunakan Notepad bawaan Windows, penggunaan text editor khusus akan memberikan banyak kemudahan seperti:

  • Pewarnaan sintaks (Syntax Highlighting)
  • Auto Complete
  • Auto Closing Tag
  • Pencarian dan Penggantian Teks
  • Manajemen File yang Lebih Baik

Beberapa Text Editor Populer:

1. Visual Studio Code

Visual Studio Code atau VS Code merupakan text editor yang paling populer saat ini.

Kelebihan:

  • Gratis
  • Ringan
  • Banyak ekstensi
  • Mendukung berbagai bahasa pemrograman

Cocok untuk:

  • Pemula
  • Mahasiswa
  • Profesional

2. Sublime Text

Sublime Text dikenal karena performanya yang cepat dan ringan.

Kelebihan:

  • Ringan
  • Cepat
  • Antarmuka sederhana

3. Notepad++

Notepad++ merupakan editor ringan yang banyak digunakan di Windows

Kelebihan:

  • Gratis
  • Mudah digunakan
  • Cocok untuk pemula

4. Notepad

Notepad adalah editor bawaan Windows.

Kelebihan:

  • Tidak perlu instalasi
  • Sangat sederhana

Kekurangan:

  • Tidak memiliki fitur khusus untuk pemrograman

Rekomendasi untuk pemula

Untuk mengikuti tutorial di sini, disarankan menggunakan Visual Studio Code (VS Code) karena memiliki banyak fitur yang mempermudah proses belajar HTML.



2.3. Struktur File Website Sederhana

Pada materi sebelumnya di Pengenalan HTML kita sudah belajar cara membuat dokumen HTML, dan kita sudah punya satu folder dengan nama belajar-html dan file index.html di dalamnya.

Sekarang buat 2 file lagi dengan nama profil.html dan kontak.html. Dengan begitu struktur dari file website kita jadi seperti ini:

belajar-html/
├── index.html
├── profil.html
└── kontak.html

File Fungsi
index.html Halaman utama
profil.html Halaman profil
kontak.html Halaman kontak


2.4. Struktur dengan Folder Gambar

Ketika website mulai berkembang, sebaiknya file dikelompokkan berdasarkan jenisnya.

Contoh:

belajar-html/
│
├── index.html
│
├── pages/
│   ├── profil.html
│   └── kontak.html
│
└── images/
    ├── logo.png
    └── banner.jpg

Keuntungan:

  • Lebih rapi
  • Mudah dikelola
  • Memudahkan pengembangan website


2.5. Mengenal Developer Tools Browser

Developer Tools atau DevTools adalah fitur bawaan browser yang digunakan untuk membantu pengembangan website.

Melalui DevTools, developer dapat:

  • Melihat struktur HTML
  • Melihat kesalahan website
  • Menguji perubahan kode
  • Menganalisis performa halaman

Cara Membuka Developer Tools

Untuk membuka DevTools kadang setiap browser punya cara yang berbeda. Untuk Google Chrome caranya adalah dengan menekan tombol F12 atau kombinasi tombol CTRL + SHIFT + I

Untuk mencoba DevTools, terlebih dahulu sesuaikan dulu struktur file website seperti pada Materi 2.4, lalu isi file index.html dengan kode berikut:

<!doctype html>
<html>
  <head>
    <title>Website Pertama Saya</title>
  </head>
  <body>
    <h1>Selamat Datang</h1>

    <p>Ini adalah website pertama saya menggunakan HTML.</p>
  </body>
</html>

Setelah itu buka di browser dan akses DevTools seperti cara yang sudah dijelaskan.

Gambar 2.1 : tampilan DevTools


Bagian penting DevTools adalah:

  • Elements : menampilkan struktur HTML dalam satu halaman.
  • Console : menampilkan informasi, pesan, dan error dari website. Meskipun kita belum menggunakan JavaScript, namun console tetap berguna untuk melihat peringatan atau kesalahan.
  • Network : melihat file yang dimuat oleh browser. Misalnya: HTML, Gambar, maupun Video.
  • Application : menampilkan informasi penyimpanan browser seperti: Cookies, Local Storage, dan Session Storage.

Hampir semua web developer menggunakan DevTools setiap hari untuk memeriksa struktur HTML, menemukan kesalahan, menguji tampilan website. Karena itu kita biasakan membuka DevTools saat belajar membuat website.
